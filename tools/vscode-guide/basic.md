# vscode 使用指南



## 文件打开模式

- 鼠标单击打开 vscode 的文件 --> preview mode

  文件名显示斜体，单机打开下一个预览文件时，当前标签页会被直接替换

- 鼠标双击打开 vscode 的文件 --> standard mode

  文件名显示正体，会一直占据一个标签位，直到手动关闭

## multi folder

当我们希望在 vscode 左侧边栏目中查看和管理多个互不隶属的文件夹，我们可以使用 Multi-Root Workspaces（多根工作区） 功能。

操作：

1. 点击顶部菜单栏的 File（文件）。
2. 选择 Add Folder to Workspace...（将文件夹添加到工作区...）。
3. 在弹出的远程路径选择框中，选择你的第二个文件夹（比如 `knowledge-base`）。

此时，你会发现左侧的资源管理器（Explorer）中同时出现了两个项目，你可以像操作同一个项目一样在它们之间切换、搜索或拖拽文件。

保存工作区状态

当你添加了多个文件夹后，VS Code 实际上是创建了一个临时的工作区。为了方便下次直接进入：

1. 点击 File > Save Workspace As...。
2. 将其保存为一个后缀为 `.code-workspace` 的文件（建议保存在服务器的家目录或项目根目录）。
3. 下次连接时：直接通过 VS Code 打开这个 `.code-workspace` 文件，它会自动建立 SSH 连接并同时加载这两个项目。

### 3. 多项目模式的进阶技巧

在远程环境下使用多根工作区，有几个非常实用的功能：

- 跨项目全局搜索：按下 `Ctrl + Shift + F` 进行搜索时，搜索范围会覆盖工作区内的所有文件夹。
- 独立配置：你可以为 `website` 设置一套 Python 解释器（比如用于你的 uv 环境），而为 `knowledge-base` 设置另一套插件或格式化规则。
- 右键管理：如果你想移除其中一个，直接在左侧目录树的项目名上点击右键，选择 Remove Folder from Workspace。
