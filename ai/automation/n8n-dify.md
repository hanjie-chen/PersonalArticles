# n8n

n8n 更像是一个整条业务 workflow 的编排平台。官方定义里，workflow 就是“由多个节点连接起来，用来自动化一个过程”的东西；这些节点可以是普通节点，也可以是 AI 相关节点。

在 n8n 里，一个节点完全可以只是一个 **纯 LLM API 调用**，也可以是更复杂的 **AI Agent 节点**；同时也可以是 HTTP、Webhook、IF、Schedule、数据库、通知这类完全非 AI 的节点。n8n 官方文档本身就把它定位成“把 AI 能力和业务流程自动化结合起来”的 workflow automation tool。

# dify

n8n 诞生于 2019 年，是典型的低代码自动化工具；而 Dify 诞生于 2023 年，完全是为了适配大模型（LLM）爆发而生的。

如果把 n8n 比作一个“全能工具箱”，那么 Dify 更像是一个“AI 实验室”。

## Dify 到底是什么？

可以把 Dify 理解为一个 AI 应用的中台。它帮你把原始的、光秃秃的大模型（比如 GPT-4），包装成一个有知识、有手脚、能对话的“智能体（Agent）”。

它主要解决的是大模型在实际使用中的三个痛点：

### 1. RAG (知识库) —— 给 AI 塞一本“参考书”

原生的 AI 只知道它训练时学到的知识。如果你问它“我公司的报销流程是什么？”，它会一本正经地胡说八道。

- Dify 的作用： 你把公司的 PDF、Notion 丢进去，Dify 会自动把它们切碎、存进数据库。
- 效果： 当有人提问时，Dify 会先去这堆文档里“翻书”，把找到的内容喂给 AI，让 AI 根据你的文档回答。这就是 RAG（检索增强生成）。

### 2. Prompt Orchestration (提示词编排) —— 给 AI 设“人设”

在 Dify 里，你可以非常方便地调试 Prompt。

- 比如你写：“你是一个资深的 DevOps 工程师，语气要简练。”
- Dify 提供了一个非常直观的界面，让你一边写 Prompt，一边在右侧测试 AI 的反应，而不需要在代码里拼字符串。

### 3. Agent (智能体) —— 给 AI 装上“手和脚”

这是 Dify 最核心的玩法。普通的 AI 只能聊天，但 Agent 模式下的 AI 可以调用工具。

- 例子： 你问“帮我查一下 GitHub 上那个仓库的最新 commit 内容”。
- Dify 的逻辑： AI 发现自己不懂，但它看到你给它配了一个“GitHub 查询工具”，它就会主动去调用那个 API，拿到结果后再分析给你听。

## n8n 与 Dify 在“编排”上的本质区别

虽然两个都有连线界面，但它们思考的方式不一样：

- n8n 的连线是“确定性”的：

  第一步做什么，第二步做什么，路径是死板的。就像工厂的流水线，A 必须流向 B。

- Dify 的连线是“启发性”的：

  在 Dify 的 Workflow 模式里，你甚至可以放一个 “回答（Answer）” 节点。你告诉 AI 几个可选的工具，AI 在运行时会自己思考：“现在的状况，我是该直接回答，还是该去查一下 Google？”

## 总结：它能帮你做什么？

对于一个开发者，Dify 的价值在于：

1. 省去写后端代码： 以前写一个 RAG 应用要写向量数据库对接、写 PDF 解析、写 API 接口，现在 Dify 全部可视化搞定。
2. 提供 API 接口： 你在 Dify 编排好之后，它会直接给你一个标准的 API。你可以直接在你的个人博客或 App 里调用它。
3. Self-host： 既然你提到了 GitHub repo，Dify 确实支持 Docker Compose 一键部署。
