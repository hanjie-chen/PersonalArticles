# daemon process

简化守护进程标准流程如下

1. `fork()` 一个子进程 → 主进程退出 → 子进程成为“孤儿”进程（由 PID 1 管理）
2. `setsid()` → 脱离控制终端，成为新的 session leader
3. `setpgid()` → 设置自己的进程组（pgid）
4. `chdir("/")`、关闭标准输入输出等，彻底“脱离用户空间”

## 为什么守护进程要脱离控制终端？

背后的设计初衷：

守护进程（daemon）本质上是后台运行的服务，不属于任何用户交互进程。为了避免和用户登录会话有关联，它必须做到：

| 操作                    | 原因                             |
| ----------------------- | -------------------------------- |
| `fork()` 之后父进程退出 | 脱离原始 shell 的控制            |
| `setsid()`              | 创建新的会话，并**脱离控制终端** |
| `chdir("/")`            | 防止占用挂载目录                 |
| 重定向 stdin/out/err    | 不依赖任何终端输出               |
| `umask(0)`              | 自己控制文件权限                 |

### 脱离终端的关键目的是：

- 不受 Ctrl+C / Ctrl+Z 等信号干扰
- 防止登录用户退出时进程被 kill
- 作为独立服务存在，系统重启自动拉起，不依赖“谁先登录”



# ctrl+c ctrl+v

简单来说

- 如果你想彻底关掉这个程序：请用 `Ctrl + C`。
- 如果你只是想离开一下，等会还要回来：请用 `Ctrl + Z`。

## 核心区别：信号（Signals）

在 Linux 中，这两个快捷键分别向进程发送了不同的系统信号：

### 1. Ctrl + C (SIGINT)

- 含义： 中断 (Interrupt)。
- 结果： 告诉程序“立刻停止运行并退出”。
- 状态： 进程会被完全杀死（Kill），从内存和任务列表中消失。
- 适用场景： 当你运行了一个程序（比如 `ping` 或者一个写错的死循环脚本），且你不再需要它运行时使用。

### 2. Ctrl + Z (SIGTSTP)

- 含义： 终端停止 (Terminal Stop)。
- 结果： 告诉程序“暂时挂起，保持现状”。
- 状态： 进程并没有消失，而是进入了后台（Background）并处于“停止/睡眠”状态。它依然占用内存，但不再消耗 CPU。
- 适用场景： 当你正在编辑一个文件（比如使用 `vim`），突然想跳出来查个资料或运行一个简单的命令，随后还想回到刚才的状态时使用。

## 不小心用了 Ctrl + Z

如果你不小心按了 `Ctrl + Z` ，需要掌握以下几个配套命令：

1. `jobs`：查看当前后台有哪些被暂停的任务。
2. `fg` (Foreground)：将最近的一个后台任务移回前台继续运行。
3. `bg` (Background)：让被暂停的任务在后台继续运行（适合那些不需要交互的任务）。

> [!note]
>
> 如果你习惯性地用 `Ctrl + Z` 来关闭程序，你的后台会堆积大量的“僵尸”进程，浪费系统资源。你可以输入 `jobs` 看看你现在是不是已经攒了一堆任务了。
